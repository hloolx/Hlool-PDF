# hlool pdf 体验重设计方案

> 目标：把"能用的盖章工具"升级为"优雅顺手的盖章工作台"。
> 本文档基于 2026-06 的代码现状（`web/src/main.tsx` 单文件 2418 行 + Go/pdfcpu 后端）。

---

## 0. 一句话诊断

现在的前端是**"表单驱动"**的：用户在右侧十几个设置区块里填数字、按按钮，再回头看画布有没有变化。
优雅的工具是**"直接操作"**的：所有事情都发生在画布上——拖进来、按下去、拉一下手柄，面板只是选中之后的属性回显。

这一层模型不换，再怎么调配色都不会"优雅"。

---

## 1. 现状问题分析

### 1.1 信息架构：右栏是设置堆，不是工作流

右栏从上到下依次是：视图设置 → 编辑工具 → 快速放置 → 新增默认 → 印章设置 → 作用范围 → 批量盖章 → 本次生成 → 加密 → 文件名备注 → 生成按钮 → 处理任务 → 任务历史，共 13 个区块同屏堆叠。

- 最不常用的"视图设置"占据最顶部黄金位置；最重要的"生成"沉在底部（顶栏又重复了一个）。
- 无论用户当前在做什么，所有面板都同时可见、同时可点 —— 没有"此刻该做什么"的引导。
- "新增默认"和"印章设置"两组几乎一样的字段（尺寸/透明度/旋转）并排出现，用户很难理解区别。

### 1.2 交互模型：核心操作全靠表单

- 放置印章要先点"添加印章"进入放置模式，再移动鼠标点击 —— 而最自然的手势是**把印章从库里拖到页面上**。
- 改大小要去右栏输入"宽度 pt"，旋转要输入角度数字 —— 选中框没有缩放手柄、没有旋转手柄。
- 单位混乱：默认尺寸用 mm，选中后编辑用 pt，骑缝章又是 pt。办公场景用户只认 mm（公章直径 40/42/45mm）。
- 骑缝章的预览是把整张章图压扁成一个条（`sizePt * 0.18`），不是真实的分割切片 —— 预览不可信。

### 1.3 多 PDF 队列是"假"的

- 可以导入多个 PDF、可以拖拽排序，但**生成永远只处理当前选中的那一个**，排序毫无作用。
- 切换 PDF 会**清空全部已放置的印章**（还弹 confirm 吓用户）。
- 老项目 PDFQFZ 的核心价值恰恰是"选一个文件夹，一套配置批量盖完"。这是目前最大的功能退化。

### 1.4 反馈系统破碎

- 一条顶部 message 文字承担所有反馈：引导、成功、报错混在一起，一闪而过。
- 生成前有两层确认弹窗（批量摘要确认 + 生成前复核），而生成本身是无损操作（只输出新文件），确认是纯摩擦。
- 删除用原生 `window.confirm`。
- 生成完成后，下载链接出现在右栏下方折叠区 —— 整个流程的"高光时刻"（拿到结果）毫无存在感。
- 没有撤销/重做。直接操作类工具没有 Ctrl+Z 等于裸奔。

### 1.5 视觉：功能性有余，质感不足

- 70px 高的顶栏放标题+副标题，信息密度低；三栏均为白底灰边，无层次。
- PDF 页面没有纸张感（无投影、画布背景与面板几乎同色）。
- 控件是 6px 圆角的基础样式，无设计令牌、无暗色模式、无动效。
- JPG 印章白底直接糊在页面上，预览效果脏。

### 1.6 半成品 UI 暴露在界面上

- "连续页"按钮 disabled + tooltip "界面预留"；
- "白底透明化/自动裁剪"显示"待实现"按钮；
- "输出文件名备注"输入框，placeholder 写着"后端暂不支持"。

未完成的功能不应该以灰色按钮的形式向用户道歉，要么实现，要么删掉。

## 2. 设计原则

1. **直接操作优先**：能在画布上完成的，绝不放进表单。面板只做"选中后的精调"。
2. **上下文呈现**：右栏内容由当前选中对象决定（什么都没选 → 文档与输出；选中印章 → 印章属性；选中骑缝章 → 骑缝属性）。
3. **零弹窗打断**：确认弹窗全部移除，用"即时执行 + Toast + 可撤销"替代。
4. **所见即所得**：预览必须与产物一致（骑缝章真实切片、白底透明实时生效）。
5. **批量是一等公民**：配置随文件保存，一套配置可以应用到整个队列。
6. **单位统一为 mm**：界面只出现毫米，pt 只存在于数据层。

### 各区域职责

| 区域 | 内容 | 替代现状的什么 |
|---|---|---|
| 顶栏 48px | 文件切换下拉（即多 PDF 队列）、撤销/重做、缩放、主题、**生成（分裂按钮：生成当前 / 生成全部）** | 70px 顶栏 + 左栏 PDF 队列 + 右栏底部生成按钮 |
| 左栏上部·页面缩略图 | 每页缩略图 + 角标（普通章数量、骑缝章色条）；点击跳页；Ctrl/Shift 多选作为页面范围 | "第 N / M 页"数字输入 + 范围文本框 |
| 左栏下部·印章架 | 常驻印章库，拖拽即放置；每个章可设别名和默认尺寸 | 左栏印章库 + "选中并放置"小按钮 |
| 中央画布 | 连续滚动页面流、纸张投影、直接操作（拖/缩/旋/吸附） | 单页 canvas + 翻页按钮 |

| 右下角浮层 | 任务进度 Toast → 完成后变结果卡（预览缩略图 + 下载 + 历史入口） | 处理任务/任务历史常驻面板 + 两层确认弹窗 |

任务历史收进一个抽屉（点结果卡上的"历史"或顶栏图标打开），不再常驻占位。

## 4. 核心交互重设计

### 4.1 放置印章：拖拽即放

- 从印章架按住一个章 → 拖到页面上 → 松手即放置。拖动过程中显示半透明跟随影像和中心十字线。
- 保留点击放置作为补充：单击印章架中的章 = "上膏"状态（光标变成印章影像），在页面点一下盖一个，Esc 退出。这天然支持"多章连盖"。
- 数据层早已支持每个 placement 独立 stampId（`Placement.stampId`），UI 放开后即可混用多个不同印章。

### 4.2 选中态：手柄 + 浮动工具条

选中印章后：

```
        ↻  ← 旋转手柄（拖动旋转，按住 Shift 吸附 15°）
   ┌────●────┐
   ●  印章    ●   ← 四角手柄等比缩放（实时显示 42mm）
   └────●────┘
   ┌──────────────────────────┐
   │ 42mm ▾ │ ⟲0° │ ◐100% │ 🗑 │  ← 浮动迷你工具条
   └──────────────────────────┘
```

- 四角手柄等比缩放，缩放时在框旁实时显示毫米数；尺寸下拉提供常用预设 40 / 42 / 45mm。
- 旋转手柄拖动旋转，Shift 吸附 15° 刻度。
- 浮动工具条覆盖 90% 的调整需求，右栏检查器做剩下 10%（精确数值、应用到范围）。
- 右键菜单：复制到全部页 / 应用到范围… / 居中 / 删除。

### 4.3 智能参考线与吸附

拖动时显示 Figma 式参考线：页面水平/垂直中线、页边距线（默认 20mm）、与同页其他印章的对齐线。靠近 4px 内自动吸附，按住 Alt 临时禁用。

这是"优雅感"性价比最高的一项——成本一个几何计算模块，收益是每次拖动都"手感很贵"。

### 4.4 页面范围：缩略图即选择器

- 缩略图支持 Ctrl 点选 / Shift 连选，选中页高亮。
- 右栏"应用到范围"默认作用于缩略图选中页；同时保留文本表达式输入（`1,3-5`），两者双向同步：输入表达式 → 缩略图实时高亮；点缩略图 → 表达式自动生成。
- 范围不再是三个 segmented 按钮 + 一个文本框的组合，而是一个带预览的输入：`[全部 ▾] 1-12, 15  （共 13 页，缩略图已高亮）`。

### 4.5 骑缝章：在页面边缘直接操作

- 开启骑缝章后，页面右缘（或所选边）出现**真实切片**的预览条 —— 前端用 canvas 按与后端相同的算法（页数/最大分割数）切出当前页对应的那一片显示出来。预览即产物。
- 切片条可以**直接上下拖动**调整位置（同步右栏的位置百分比），可拖动端点调整尺寸。
- 缩略图栏在启用范围内的每页右缘显示一条红色细带，直观看到"哪些页参与骑缝"。
- 右栏检查器：边（四个图标按钮）、页面范围、尺寸 mm、分割数（默认"自动 = 页数"，可设上限）、透明度。

### 4.6 批量盖章：九宫格定位器

替换现在的 `<select>`（页面居中/左上/右上…）：

```
┌───┬───┬───┐
│ ◉ │   │   │   九宫格点选位置
├───┼───┼───┤   + 边距 mm（x / y）
│   │   │   │   + ☑ 轻微随机（偏移 ±1mm，旋转 ±2°）
├───┼───┼───┤
│   │   │ ◯ │
└───┴───┴───┘
```

选择后画布上所有目标页缩略图实时显示预计位置，不再需要"预览批量摘要"确认弹窗 —— 写入后本来就可以逐个调整或 Ctrl+Z。

### 4.7 多 PDF：配置随文件，生成全部

- 每个 PDF 的 placements / 骑缝配置以 `fileId` 为键保存在前端 store（并镜像到 localStorage），**切换文件不再丢配置、不再弹确认**。
- 顶栏文件下拉里每个文件显示配置状态点（● 已配置 / ○ 未配置）和页数。
- 新增"**将当前配置应用到队列其他文件**"：按相对规则复制（位置语义为"右下角 + 边距"，自动适配不同页面尺寸；逐页绝对坐标的配置按页码对应复制，超出页数的截断）。
- 生成按钮变分裂按钮：`⚡生成当前` / 下拉 `⚡生成全部（3 个文件）`。生成全部 = 顺序创建 N 个任务，进度浮层显示队列总进度，完成后结果卡支持"全部下载（zip）"（后端可选支持，初期逐个下载也可）。

这一条直接补齐对 PDFQFZ "文件夹批量"的功能对照。

### 4.8 生成与结果：零弹窗闭环

1. 点生成 → 立即创建任务（不再弹"生成前复核"），按钮原位变进度环。
2. 右下角浮层显示进度；失败显示中文原因 + "重试"。
3. 完成 → 浮层变**结果卡**：输出文件首页缩略图（PDF.js 渲染结果文件）、文件名、`[下载] [预览] [历史]`。预览在画布上方以浮层打开结果 PDF，可翻页核对后再下载。
4. "复核"需求改由生成按钮的下拉面板承担：悬停/展开即可看到 `3 个普通章 · 骑缝章右侧 1-12 页 · AES-256 加密 · 输出名 合同2026-已盖章.pdf`，一目了然但不挡路。

### 4.9 密码：按需出现

- 删除左栏常驻的"PDF 打开密码"输入框。上传时后端若返回"需要密码"，原地弹出小输入框重试。
- 输出加密移到右栏"输出设置"，默认折叠。

### 4.10 键盘与撤销

| 快捷键 | 行为 |
|---|---|
| Ctrl+Z / Ctrl+Shift+Z | 撤销 / 重做（placements、骑缝配置全量入栈） |
| 方向键 / Shift+方向键 | 微移 0.5mm / 5mm |
| Delete | 删除选中 |
| Ctrl+D | 复制选中印章（偏移 5mm） |
| Esc | 退出放置模式 / 取消选中 |
| PgUp/PgDn、Home/End | 翻页 |
| Ctrl+滚轮 | 缩放（以光标为中心） |
| Ctrl+Enter | 生成 |

## 5. 视觉系统

### 5.1 设计令牌（CSS Variables，支持暗色）

```css
:root {
  /* 主色：印泥红 —— 域感强，克制使用（主按钮/选中态/骑缝色带） */
  --accent: #C8372D;
  --accent-hover: #B02E25;
  --accent-soft: #FBEAE8;

  --bg-canvas: #ECEFF2;   /* 画布区比面板深一档，纸张才能"浮"起来 */
  --bg-panel: #FFFFFF;
  --bg-sunken: #F6F7F9;
  --border: #E2E6EA;
  --text: #1D2329;
  --text-muted: #6B7680;

  --radius-md: 8px;  --radius-sm: 6px;
  --shadow-page: 0 1px 2px rgb(16 24 40 / .07), 0 12px 32px rgb(16 24 40 / .10);
  --shadow-pop: 0 4px 16px rgb(16 24 40 / .14);
  --font-ui: 13px/1.5 Inter, "Segoe UI", "Microsoft YaHei", system-ui;
}
[data-theme="dark"] {
  --accent: #E05545;
  --bg-canvas: #16191D; --bg-panel: #1F2329; --bg-sunken: #181B20;
  --border: #2C323A; --text: #E6EAEE; --text-muted: #8B95A0;
}
```

- 数字一律 `font-variant-numeric: tabular-nums`（尺寸、页码、百分比不跳动）。
- 区块标题改 11px 大写灰字（`PAGES` 风格的克制标签），不再用 `<h2>` 大黑字。

### 5.2 画布质感

- 画布背景 `--bg-canvas`，页面白纸 + `--shadow-page` 投影 + 1px 边线，页与页间距 24px。
- 印章选中框用 accent 色 1.5px 边 + 四角圆点手柄；未选中 hover 时显示 1px 虚线提示可点。
- 印章库缩略图背后铺透明棋盘格，立刻能看出哪张图带透明通道。

### 5.3 动效规范

- 统一两档：交互反馈 120ms ease-out（hover/选中/手柄），空间变化 200ms cubic-bezier(0.2, 0, 0, 1)（面板、抽屉、Toast）。
- **盖章微动效**（域专属的点睛之笔）：印章落下时 `scale 1.08 → 1` + 轻微透明度沉降，160ms，像真的按了一下印泥。
- 任务完成时结果卡从右下滑入 + 生成按钮短暂变绿勾。

### 5.4 文案规范

- 引导用空状态而不是常驻顶栏消息："拖入 PDF 开始" → 加载后画布角落出现一次性提示"从下方印章架拖一个章到页面上"。
- Toast 动词开头、一句话：`已放置到第 3 页`、`已应用到 12 页`、`生成完成 · 2.4MB`。
- 错误说人话并给出下一步：`这份 PDF 有打开密码 → [输入密码]`。

## 6. 工程落地

### 6.1 前端结构拆分（告别 2418 行单文件）

```
web/src/
  main.tsx                 # 仅挂载
  app/App.tsx              # 布局骨架
  state/
    store.ts               # zustand：文档/印章/placements（按 fileId 分桶）
    history.ts             # zundo 撤销中间件
    persistence.ts         # localStorage 镜像
  features/
    viewer/                # 连续页渲染（IntersectionObserver 懒渲染）、缩放
    thumbnails/            # 缩略图栏（PDF.js 0.2x 渲染 + 缓存）
    placements/            # StampBox、手柄、参考线、拖放、快捷键
    seam/                  # 切片预览（与后端同算法）、边缘拖动
    stamps/                # 印章架、白底透明化（canvas 阈值+羽化）、WebP 转 PNG
    jobs/                  # 进度浮层、结果卡、历史抽屉
    inspector/             # 上下文检查器
  ui/                      # Button/Input/Slider/Segmented/Toast/Popover…
  lib/                     # units.ts(mm⇄pt)、pages.ts(范围表达式)、geometry.ts(吸附)
```

### 6.2 技术选型建议

| 事项 | 建议 | 理由 |
|---|---|---|
| 样式 | Tailwind v4 + 上述令牌（或保留手写 CSS + variables） | 现有 CSS 已 1130 行且无令牌；Tailwind 配合组件拆分最快达到一致性 |
| 基础组件 | Radix Primitives（Dialog/Popover/Slider/Tooltip/DropdownMenu）或 shadcn/ui | 原生 confirm/select 全部替换，键盘可达性白送 |
| 状态 | zustand + zundo | 撤销/重做 10 行接入；跨组件共享比 prop 钻取干净 |
| 拖拽手势 | @use-gesture/react | 统一处理拖动/捏合缩放/手柄旋转 |
| PDF 渲染 | 维持 pdfjs-dist，加 ImageBitmap 缩略图缓存 | 不需要换 |

### 6.3 后端需要的小改动（按优先级）

1. **自定义输出文件名**：`createJobRequest` 加 `outputName`，下载时写入 `Content-Disposition`（含 UTF-8 编码与脱敏）。→ 解锁文件名模板 `{原名}-已盖章.pdf`。
2. **结果内联预览**：下载接口支持 `?inline=1`（`Content-Disposition: inline`），前端用 PDF.js 渲染结果卡缩略图与预览浮层。
3. **WebP 印章**：上传时用 `golang.org/x/image/webp` 解码转 PNG 存储；前端所有"WebP 不支持"的提示文案删除。（也可先在前端 canvas 转码，后端不动。）
4. **加密 PDF 探测**：上传无密码的加密 PDF 时返回结构化错误码 `password_required`，支撑 4.9 的按需密码弹层。
5. **批量任务（可后置）**：初期前端循环 `POST /api/jobs` 即可（受 `HLOOL_MAX_JOBS` 排队）；后续可加 `POST /api/jobs/batch` + zip 打包下载。
6. **草稿持久化（可选）**：`PUT /api/files/{id}/draft` 保存盖章配置，跨设备/重启恢复；初期 localStorage 已够。

## 附：验收标准（什么叫"优雅了"）

1. 新用户不看任何说明，30 秒内完成"导入 → 盖一个章 → 生成下载"。
2. 全程零弹窗、零原生 confirm；任何误操作 Ctrl+Z 可回退。
3. 预览与产物一致：骑缝切片、白底透明、旋转角度在画布上看到什么，输出就是什么。
4. 10 个文件的批量盖章，总操作次数 ≤ 单文件的 1.5 倍。
5. 界面上不存在任何 disabled 的"待实现"元素。
